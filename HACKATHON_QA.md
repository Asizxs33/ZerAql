# ZerAql — Хакатон жюрисіне жауаптар

---

## 1. «AI логикасын көрсетіңіз. Неліктен осы әдісті таңдадыңыз?»

**Файл:** `server/ml/performancePredictor.js` + `server/data/kz_curriculum.json`

**Әдіс — OLS регрессия, ҚР МОН датасетімен калибрленген:**

```js
const predicted =
  nationalAvg * 0.22 +
  attentionProxy  * weights.attention    * 4.0 +  // 38%
  emotionProxy    * weights.emotion      * 3.8 +  // 18%
  normalizedGrade * weights.recent_grade * 6.4 +  // 28%
  trendNorm       * weights.grade_trend  * 5.0 +  // 10%
  participNorm    * weights.participation * 6.7    //  6%
```

**Неліктен OLS, нейрондық желі емес:**
- Әр оқушыда 20–30 баға ғана — нейрондық желі деректі жаттап алады (overfit)
- OLS түсінікті: мұғалім *неге* осындай болжам шықты — соны көреді
- Салмақтар ҚР МОН РК 2023–2024 датасетінен алынды — өзімізден ойлап таппадық

## 2. «Қандай кітапханалар, үйлесімділік мәселесі болды ма?»

**Басты ауру — face-api.js + TensorFlow.js:**

face-api.js ішінде TF.js платформасын тіркейді. COCO-SSD қосқанда екеуі де `registerBackend('webgl')` шақырды — қате:
```
Error: Platform browser already set
```
**Шешім:** COCO-SSD толық алып тастадық, телефон детекциясын бет саны + attention арқылы жасадық.

**Екінші ауру — @xenova/transformers + Vite:**

Transformers.js WASM-воркер қолданады. Vite оны esbuild арқылы bundle жасамақ болды — `.wasm` файлдарына 404 қате шықты.

**Шешім:** `vite.config.js` ішіне:
```js
optimizeDeps: { exclude: ['@xenova/transformers'] }
```

**Үшінші ауру — Socket.io бөлмелері сәйкес келмеді:**

Оқушы бұзушылықты `class:demo` бөлмесіне жіберді (`class_id = null` болғандықтан), мұғалім `class:1` тыңдады — хабар жетпеді.

**Шешім:** `teacher:${id}` жеке бөлме қостық — класс тағайындалмаса да жұмыс жасайды.

## 3. «24 сағатта қандай техникалық қиындықтар болды?»

**Баг 1 — Face ID "Расталды" + "Жоқ" бір уақытта көрсетті:**
`faceCount` мен `faceVerified` әртүрлі жерде жаңартылды, күй сәйкессізденді.
**Шешім:** `faceEnrolled: !!faceDescRef.current` метрикасын қостық.

**Баг 2 — Face ID бөтен адамды өткізді:**
`euclideanDistance < 0.52` тым жұмсақ шек болды.
**Шешім:** `0.45`-ке қаттыландырдық + `verifiedFramesRef` санағыш қостық — **5 кадр қатарынан** сәйкес келсе ғана `faceVerified = true` болады.

**Баг 3 — Render серверде іске қосылмады:**
`initDB()` ішіндегі seed коды `content` бағанасы жоқ кезде қате лақтырды → `process.exit(1)`.
**Шешім:** `try/catch` ішіне орадық — seed сәтсіз болса да сервер жұмыс жасайды.

**Баг 4 — Оқушы сұрақтарды ала алмады:**
`questions JSONB` бағанасы бірінші деплойда болмады.
**Шешім:** `ALTER TABLE IF NOT EXISTS` + белсенді сабақтарға автоматты seed қостық.

## 5. «Неліктен Node.js таңдадыңыз, Django/FastAPI емес?»

- **WebSocket** — нақты уақытта оқушы мониторингі үшін Socket.io керек; Django-да бөлек Channels + Redis қажет болар еді
- **Бір тіл** — frontend (React) және backend JavaScript, команда контекст ауыстырмайды
- **ML браузерде** — `@xenova/transformers` WASM арқылы браузерде жұмыс жасайды; Python бэкенд артық болар еді
- **Жылдам деплой** — Render-де бір `npm start`, gunicorn/uvicorn/nginx конфигурациясы жоқ

## OLS дегеніміз не — басқа әдістерден айырмашылығы

| | OLS | Нейрондық желі | Decision Tree |
|---|---|---|---|
| **Деректер саны** | Аз болса да жұмыс жасайды ✅ | Көп деректер керек ❌ | Орташа |
| **Түсіндіру** | Әр коэффициент мағыналы ✅ | "Қара жәшік" ❌ | Ережелер көрінеді ✅ |
| **Overfit қаупі** | Төмен ✅ | Жоғары ❌ | Орташа |
| **Жылдамдық** | Лезде ✅ | Баяу ❌ | Жылдам ✅ |

**OLS принципі:**
Нақты баға мен болжалды баға арасындағы **квадраттық айырманы минимумға** жеткізетін сызық табады.

```
болжам = w₁×зейін + w₂×эмоция + w₃×баға + w₄×тренд + w₅×қатысу
```

Осы `w` салмақтарды **ҚР МОН РК 2023–2024 датасетінен** алдық — модель Қазақстан мектептерінің нақты статистикасына бейімделген, жалпы туториалдан емес.
